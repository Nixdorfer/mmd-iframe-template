欢迎来到Nix的iframe自动注入模板仓库

# 仓库结构

该仓库由多个不同的项目组成 其中

- [template](./template)文件夹中存储着可以通过[KEY:k1=v1;kx=vx]被正则捕获并自定义的模板型项目

- [instance](./instance)文件夹中存储着为某项目特化形成的项目

- [test](./test)文件夹中存储着用于测试新功能的文件 一般而言 您不会需要查看该文件

# 项目结构

每个模板型项目都包含以下三个文件

- README.md 为该项目的可用参数说明 正则表达式也在这份文件内

- inject.html 为该项目的项目相关正则替换内容 直接粘贴进去即可

- index.html 为该项目的模板 一般而言 您不会需要查看该文件

- src 为该模板调用的图像资源文件夹 其中可能会含有若干个图像

# 参数结构

对于大部分项目而言 您可以通过注入参数来调整实际显示内容 格式一般为 `[KEYWORD:k1=v1;k2=v2;kx=vx]` 这样的键值对 并且一般而言会使用正则表达式 `/\[KEYWORD:([^\]]+)\]/`

在单个 `[KEYWORD:k1=v1;k2=v2;kx=vx]` 内不允许换行 这会导致MMD捕获失败 但是如果您需要有换行内容 可以通过键入换行标记 `\n` 来指导程序在此处换行渲染
